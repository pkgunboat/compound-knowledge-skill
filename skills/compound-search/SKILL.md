# /compound-search — 知识检索

## 目标

从全局知识库中检索与当前问题相关的历史经验，避免重复踩坑。

## 使用场景

- 遇到报错，想查查之前是否解决过类似问题
- 开始新任务前，查看是否有相关的模式/最佳实践
- 调试卡住时，看看历史上类似问题的根因

## 执行流程

### Step 1: 确定搜索策略

从用户描述或当前对话上下文中提取关键信息：
- 错误信息关键词
- 涉及的技术栈/工具
- 问题分类（对应 solutions 的子目录）

### Step 2: 多维度搜索

依次执行以下搜索，合并结果：

1. **按分类目录浏览**
   - 如果能判断问题分类，直接 `Glob("~/.claude/knowledge/solutions/[category]/*.md")` 列出该分类下的所有文档
   - 读取文件的 frontmatter 中的 title 和 tags 做初步筛选

2. **按关键词搜索**
   - 用 Grep 在 `~/.claude/knowledge/solutions/` 中搜索错误信息、关键函数名、技术名词
   - 搜索范围包括正文和 tags

3. **按标签搜索**
   - 用 Grep 搜索 `tags:` 行中匹配的标签

### Step 3: 排序与呈现

将匹配的文档按相关度排序，输出：

```
找到 N 份相关经验:

1. [标题] ⟵ 高度相关
   文件: [路径]
   类型: [bug-track / knowledge-track]
   标签: [tags]
   创建: [日期]
   匹配原因: [为什么认为相关]

2. [标题] ⟵ 可能相关
   ...
```

### Step 4: 展示详情

如果用户感兴趣，读取并展示完整文档内容。

如果找到高度相关的文档，主动提示：
- "这份文档记录了类似问题，建议先看看之前的根因分析和解决方案"
- "注意：这份文档创建于 [日期]，相关引用可能已过时，建议验证后再使用"

### Step 5: 无结果处理

如果没有找到相关文档：
```
未找到相关经验。解决当前问题后，建议使用 /compound 沉淀本次经验。
```

## 关键原则

1. **宁多勿漏** — 搜索时放宽条件，把可能相关的都列出来，由用户判断
2. **标注时效** — 对较旧的文档，提醒用户验证内容是否仍然适用
3. **引导沉淀** — 搜索无结果时，引导用户在解决问题后使用 /compound 记录
